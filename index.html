LCM — Least Common Multiple

Math Formula-based
▼

The Least Common Multiple (LCM) of two integers a and b is the smallest positive integer that is exactly divisible by both a and b.

Key Properties:

  • LCM(a, b) = (a × b) / GCD(a, b) — the most efficient formula
  • LCM(a, b) = LCM(b, a) — commutative property
  • LCM(n, 1) = n for any integer n
  • If one number divides the other, LCM = the larger number

Use Cases: Synchronization problems, finding common denominators in fractions, scheduling tasks (when do two recurring events next align).

Overflow Warning When computing a × b, large values can overflow. Use a // GCD(a,b) * b instead to divide first.

Problem: Find LCM(12, 18).

Step 1: Find GCD(12, 18) using Euclidean algorithm
  → 18 = 12 × 1 + 6  // remainder = 6
  → 12 = 6 × 2 + 0  // remainder = 0, stop
  → GCD(12, 18) = 6

Step 2: Apply formula
  → LCM = (12 × 18) / 6 = 216 / 6 = 36

Verify: 36 / 12 = 3 ✓   36 / 18 = 2 ✓

LCM via GCD

Always use a*b//gcd(a,b) — never list multiples manually.

LCM of Array

Chain it: LCM(a,b,c) = LCM(LCM(a,b), c) using reduce.

Overflow-safe

Compute a // gcd(a,b) * b to avoid intermediate overflow.

Python
import math
from functools import reduce

def lcm(a, b):
    return a * b // math.gcd(a, b)

# LCM of an entire list
def lcm_list(nums):
    return reduce(lcm, nums)

# Example
print(lcm(12, 18))       # 36
print(lcm_list([4,6,10])) # 60

GCD — Greatest Common Divisor

MathEuclidean Algorithm
▼

The Greatest Common Divisor (GCD) of two integers is the largest positive integer that divides both without a remainder. Also called HCF (Highest Common Factor).

Key Properties:

  • GCD(a, 0) = a — base case
  • GCD(a, b) = GCD(b, a % b) — Euclidean algorithm recurrence
  • GCD(1, n) = 1 for any n (they are coprime)
  • GCD(a, a) = a

Use Cases: Simplifying fractions, checking coprimality, RSA cryptography, solving linear Diophantine equations.

Time Complexity The Euclidean algorithm runs in O(log(min(a,b))) time — very fast even for huge inputs.

Problem: Find GCD(48, 18) step by step.

GCD(48, 18):
  → 48 = 18 × 2 + 12  → GCD(18, 12)
  → 18 = 12 × 1 + 6  → GCD(12, 6)
  → 12 = 6 × 2 + 0  → GCD(6, 0) = 6

Answer: GCD(48, 18) = 6
Python
# Iterative Euclidean (preferred)
def gcd(a, b):
    while b:
        a, b = b, a % b
    return a

# Recursive version
def gcd_rec(a, b):
    return a if b == 0 else gcd_rec(b, a % b)

# Check if two numbers are coprime
def coprime(a, b):
    return gcd(a, b) == 1
#1071 Greatest Common Divisor of Strings Easy

Return the largest string x that divides both str1 and str2 (both are made by repeating x).

Python
import math

def gcdOfStrings(str1, str2):
    # If no divisor exists, concatenations differ
    if str1 + str2 != str2 + str1:
        return ""
    g = math.gcd(len(str1), len(str2))
    return str1[:g]

If a string x divides both, then str1+str2 == str2+str1. The GCD of their lengths gives the length of the largest divisor string.

Time: O(n+m) Space: O(n+m)

Primes, Modular Arithmetic & Bit Tricks

SieveModularBitwise
▼

Sieve of Eratosthenes: Efficiently find all primes up to N. Mark all multiples of each prime as composite. Time: O(n log log n).

Modular Arithmetic: Essential to prevent overflow in competitive programming.

  • (a + b) % m = ((a%m) + (b%m)) % m
  • (a * b) % m = ((a%m) * (b%m)) % m

Fast Exponentiation (Binary Exponentiation): Computes a^b mod m in O(log b) instead of O(b).

Bit OperationCodeUse Case
Check evenn & 1 == 0Parity
Power of 2n & (n-1) == 0Is power of 2?
Set bit in | (1 << i)Set specific bit
Clear bit in & ~(1 << i)Clear specific bit
XOR swapa ^= b; b ^= a; a ^= bSwap without temp
Python
# Sieve of Eratosthenes — O(n log log n)
def sieve(n):
    is_prime = [True] * (n + 1)
    is_prime[0] = is_prime[1] = False
    for i in range(2, int(n**0.5) + 1):
        if is_prime[i]:
            for j in range(i*i, n+1, i):
                is_prime[j] = False
    return [i for i in range(n+1) if is_prime[i]]

# Fast Power — a^exp mod m in O(log exp)
def power(base, exp, mod):
    result = 1
    base %= mod
    while exp > 0:
        if exp % 2 == 1:
            result = result * base % mod
        base = base * base % mod
        exp //= 2
    return result

def build_prefix(arr):
    prefix = [0] * (len(arr) + 1)

def range_sum(prefix, l, r):  # inclusive [l, r]

def maxSubArray(nums):

#560Subarray Sum Equals KMedium

Count contiguous subarrays that sum to k. Must run in O(n).

Python
def subarraySum(nums, k):
    count, prefix_sum = 0, 0
    freq = {0: 1}  # prefix_sum → frequency
    for num in nums:
        prefix_sum += num
        count += freq.get(prefix_sum - k, 0)
        freq[prefix_sum] = freq.get(prefix_sum, 0) + 1
    return count

Key insight: if prefix[j] − prefix[i] = k, subarray [i+1..j] sums to k. Use hashmap to track prefix frequency.

Time: O(n)Space: O(n)

Linked Lists

SinglyDoublyFloyd's Algorithm
▼

A Linked List is a linear sequence of nodes where each node holds a value and a pointer to the next node. Nodes are NOT stored contiguously — no random access.

Types: Singly (one next pointer), Doubly (next + prev pointers), Circular (last → head).

OperationSingly LLDoubly LL
Access by indexO(n)O(n)
Insert at headO(1)O(1)
Insert at tail*O(1)O(1)
Delete known nodeO(n) findO(1) with prev

*O(1) only when tail pointer is maintained.

Use Cases LRU Cache (doubly LL + hashmap), undo/redo stacks, implementing stacks/queues, polynomial representation.

Floyd's Cycle Detection — slow pointer moves 1 step, fast moves 2. If cycle exists, they meet.

List: 1 → 2 → 3 → 4 → 5 → (back to 3)

Step 1: slow=1, fast=1
Step 2: slow=2, fast=3
Step 3: slow=3, fast=5
Step 4: slow=4, fast=4 ← MEET! Cycle detected ✓

Why it works: fast gains 1 step per iteration on slow.
In a cycle of length L, they meet after at most L steps.

Fast & Slow Pointers

Detect cycles, find middle. Slow moves 1x, fast moves 2x.

Dummy Head Node

Simplifies edge cases for head deletions/insertions. Always return dummy.next.

Reverse In-Place

Iterative: prev, curr, next. Three-pointer dance moving backward.

Merge Two Sorted

Two pointers picking smaller head. Used in Merge Sort on LL.

Python — Reverse Linked List
def reverseList(head):
    prev, curr = None, head
    while curr:
        nxt = curr.next
        curr.next = prev   # reverse the pointer
        prev = curr        # advance prev
        curr = nxt         # advance curr
    return prev            # new head
#141Linked List CycleEasy

Determine if a linked list has a cycle using O(1) space.

Python
def hasCycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False
Time: O(n)Space: O(1)

Stacks

LIFOMonotonic Stack
▼

A Stack follows Last In, First Out (LIFO). All operations happen at the top.

Core Operations (all O(1)): push, pop, peek/top, isEmpty.

In Python, use a list: append() = push, pop() = pop, [-1] = peek.

Use Cases Function call stack, balanced parentheses, expression evaluation, DFS (iterative), backtracking, undo/redo, monotonic stack (next greater element).

Balanced Brackets

Push open brackets, pop+match on close. Return stack empty at end.

Monotonic Stack

Maintain increasing/decreasing stack. Pops tell you "next greater/smaller" relationships.

Iterative DFS

Replace recursion with an explicit stack. Push children, pop to visit.

Two-Stack Queue

One stack for input, one for output. Transfer when output empty.

Python — Monotonic Stack
# Next Greater Element
def nextGreater(nums):
    result = [-1] * len(nums)
    stack = []  # stores indices
    for i, num in enumerate(nums):
        while stack and nums[stack[-1]] < num:
            result[stack.pop()] = num
        stack.append(i)
    return result
#20Valid ParenthesesEasy

Determine if a string of brackets is valid (balanced and properly ordered).

Python
def isValid(s):
    stack = []
    mapping = {')': '(', '}': '{', ']': '['}
    for char in s:
        if char in mapping:
            top = stack.pop() if stack else '#'
            if mapping[char] != top:
                return False
        else:
            stack.append(char)
    return not stack

Queues

FIFOBFSDeque
▼

A Queue follows First In, First Out (FIFO). Elements are added at the rear and removed from the front.

In Python use collections.deque for O(1) both-end operations. popleft() is O(1) vs O(n) for a list.

Variants: Deque (both-end O(1)), Priority Queue (heapq), Circular Queue (fixed-size OS scheduling).

BFS is the #1 use case Any shortest-path problem on unweighted graphs or grids — always reach for a queue + visited set.
Python — BFS Template
from collections import deque

def bfs(graph, start, target):
    queue = deque([(start, 0)])  # (node, distance)
    visited = {start}
    while queue:
        node, dist = queue.popleft()
        if node == target:
            return dist
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, dist+1))
    return -1  # not reachable
#994Rotting OrangesMedium

Grid of 0 (empty), 1 (fresh), 2 (rotten). Rotten oranges spread to 4-directional neighbors each minute. Return minutes until all fresh rot, or -1.

Python
from collections import deque

def orangesRotting(grid):
    rows, cols = len(grid), len(grid[0])
    queue, fresh = deque(), 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2: queue.append((r, c, 0))
            elif grid[r][c] == 1: fresh += 1
    max_time = 0
    dirs = [(0,1),(1,0),(0,-1),(-1,0)]
    while queue:
        r, c, t = queue.popleft()
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                grid[nr][nc] = 2; fresh -= 1
                max_time = max(max_time, t+1)
                queue.append((nr, nc, t+1))
    return max_time if fresh == 0 else -1

Multi-source BFS: Start all rotten oranges in queue simultaneously at time 0 — they spread in parallel.

Time: O(m×n)Space: O(m×n)

Hashing

O(1) avg lookupCollisionsHashMap / HashSet
▼

A Hash Table maps keys to values using a hash function to compute array indices. Average O(1) for insert, delete, and search.

Collision Resolution:

  • Chaining: Each bucket holds a linked list of colliding entries.

Good Hash Function: Deterministic, uniform distribution, fast to compute, minimizes collisions.

Python Built-ins dict = hash map (key→value), set = hash set (unique keys). Both O(1) average. Use collections.defaultdict and collections.Counter for common patterns.

#1Two SumEasy

Return indices of two numbers in array that add up to target.

def twoSum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        comp = target - num
        if comp in seen:
            return [seen[comp], i]
        seen[num] = i

#128Longest Consecutive SequenceMedium

Find the length of the longest consecutive sequence in O(n).

Python
def longestConsecutive(nums):
    num_set = set(nums)
    best = 0
    for n in num_set:
        if n - 1 not in num_set:  # sequence start
            length = 1
            while n + length in num_set:
                length += 1
            best = max(best, length)
    return best

Only start counting from sequence beginnings (no predecessor). Each element is visited at most twice.

Time: O(n)Space: O(n)

3.1

Recursion

Base CaseCall StackBacktracking
▼

Recursion solves problems by having a function call itself on smaller subproblems until reaching a base case.

Two essential components:

  • Base Case: Condition that stops the recursion.
  • Recursive Case: Breaks the problem smaller + recursive call.

Each call adds a stack frame. Python default limit ≈ 1000 (can raise with sys.setrecursionlimit).

Memoization = Recursion + Cache Without memo, Fibonacci is O(2^n). With @lru_cache, it becomes O(n). Always check for overlapping subproblems.

Linear Recursion

One call per step. Factorial, power, reverse string.

Binary Recursion

Two calls per step. Fibonacci, merge sort, tree traversal.

Backtracking

Choose → Explore → Unchoose. Generate all valid states.

Memoization

@lru_cache or manual dict. Cache overlapping subproblems.

Python — Backtracking Template
def backtrack(state, choices):
    if is_solution(state):
        result.append(state[:])
        return
    for choice in choices:
        state.append(choice)      # Choose
        backtrack(state, ...)     # Explore
        state.pop()               # Unchoose (backtrack)
#78SubsetsMedium

Return all possible subsets (power set) of a distinct integer array.

Python
def subsets(nums):
    result = []
    def backtrack(start, current):
        result.append(current[:])
        for i in range(start, len(nums)):
            current.append(nums[i])
            backtrack(i + 1, current)
            current.pop()  # backtrack
    backtrack(0, [])
    return result
Time: O(2^n · n)Space: O(n)
3.2

Searching

Binary SearchO(log n)Search on Answer
▼

Linear Search: O(n). Scan every element. Works on any array.

Binary Search: O(log n). Repeatedly halve search space. Requires sorted input.

For n=1,000,000: binary search needs at most 20 comparisons (log₂(10⁶) ≈ 20).

Safe Mid Calculation Use mid = left + (right - left) // 2 instead of (left + right) // 2 to avoid integer overflow in languages with fixed int size (not an issue in Python, but good habit).

Classic Binary Search

Find exact target. Return mid or -1.

Lower Bound

First position where arr[i] >= target. Move left=mid+1 when too small.

Upper Bound

First position where arr[i] > target.

Search on Answer

Binary search over the answer space when you can check feasibility.

Python — Binary Search
def binarySearch(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target:   return mid
        elif nums[mid] < target: left = mid + 1
        else:                     right = mid - 1
    return -1
#33Search in Rotated Sorted ArrayMedium

Search in a sorted array that has been rotated at some unknown pivot point.

Python
def search(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target: return mid
        if nums[left] <= nums[mid]:   # left sorted
            if nums[left] <= target < nums[mid]:
                right = mid - 1
            else: left = mid + 1
        else:                          # right sorted
            if nums[mid] < target <= nums[right]:
                left = mid + 1
            else: right = mid - 1
    return -1

At least one half is always sorted. Determine which half, then check if target falls within it.

Time: O(log n)Space: O(1)

Merge Sort — O(n log n)
def mergeSort(arr):
    if len(arr) <= 1: return arr
    mid = len(arr) // 2
    left = mergeSort(arr[:mid])
    right = mergeSort(arr[mid:])
    return merge(left, right)

def merge(L, R):
    result, i, j = [], 0, 0
    while i < len(L) and j < len(R):
        if L[i] <= R[j]: result.append(L[i]); i += 1
        else:             result.append(R[j]); j += 1
    return result + L[i:] + R[j:]
Quick Sort — O(n log n) avg
def quickSort(arr, low=0, high=None):
    if high is None: high = len(arr) - 1
    if low < high:
        p = partition(arr, low, high)
        quickSort(arr, low, p - 1)
        quickSort(arr, p + 1, high)

def partition(arr, low, high):
    pivot, i = arr[high], low - 1
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i+1], arr[high] = arr[high], arr[i+1]
    return i + 1
#75Sort Colors (Dutch National Flag)Medium

Sort an array of 0s, 1s, and 2s in-place in a single pass without extra space.

Python
def sortColors(nums):
    low, mid, high = 0, 0, len(nums) - 1
    while mid <= high:
        if   nums[mid] == 0: nums[low], nums[mid] = nums[mid], nums[low]; low += 1; mid += 1
        elif nums[mid] == 1: mid += 1
        else:               nums[mid], nums[high] = nums[high], nums[mid]; high -= 1

Three pointers: low=end of 0s, high=start of 2s, mid=explorer. When swapping with high, don't advance mid (swapped element unexamined).

Time: O(n)Space: O(1)
3.4

Dynamic Programming ⭐ Most Important

MemoizationTabulationOptimal Substructure
▼

Dynamic Programming solves problems by breaking them into overlapping subproblems and caching results to avoid recomputation.

Two required properties:

  • Optimal Substructure: Optimal solution = optimal solutions to subproblems.
  • Overlapping Subproblems: Same subproblems appear multiple times.

Two approaches:

  • Top-Down (Memoization): Recursive + cache. Natural to write. Has call stack overhead.
  • Bottom-Up (Tabulation): Iterative table filling. Faster, no recursion overhead. Build from base cases up.
DP vs Recursion vs Greedy Recursion without cache = exponential. DP = recursion with cache = polynomial. Greedy = DP's special case where the greedy choice is always optimal without needing to explore others.

The 5-Step DP Framework:

1
Define the State — What does dp[i] represent? Be precise.
2
Find the Recurrence — How does dp[i] relate to smaller states?
3
Base Cases — The smallest valid states that can be answered directly.
4
Order of Computation — Ensure smaller states are computed before larger ones.
5
Return the Answer — Which dp cell holds the final answer?
Example: Coin Change State: dp[a] = min coins for amount a. Recurrence: dp[a] = min(dp[a-coin]+1) for all valid coins. Base: dp[0]=0. Order: fill a=1 to amount. Answer: dp[amount].

Linear DP

1D state. dp[i] = dp[i-1] + dp[i-2]. Climbing Stairs, House Robber.

0/1 Knapsack

2D state: items × capacity. For each item: skip or take.

Unbounded Knapsack

Same as 0/1 but item can be reused. Use same row when iterating.

String DP (LCS)

2D table on two sequences. dp[i][j] = LCS of s1[0..i] and s2[0..j].

LIS

dp[i] = LIS length ending at index i. Look back at all j < i.

Interval DP

dp[i][j] = solve subarray [i..j]. Burst Balloons, Matrix Chain.

#70Climbing StairsEasy

You can climb 1 or 2 steps at a time. In how many distinct ways can you reach step n?

Python
def climbStairs(n):
    if n <= 2: return n
    a, b = 1, 2
    for _ in range(3, n+1):
        a, b = b, a + b  # Fibonacci
    return b

Ways to reach step n = ways from (n-1) + ways from (n-2). This is exactly Fibonacci. Space-optimized to O(1) using two variables.

Time: O(n)Space: O(1)
#322Coin ChangeMedium

Given coin denominations, find the minimum number of coins to make a target amount. Return -1 if impossible.

Python
def coinChange(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0  # base case
    for a in range(1, amount + 1):
        for coin in coins:
            if coin <= a:
                dp[a] = min(dp[a], dp[a - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1
Time: O(amount × coins)Space: O(amount)
#1143Longest Common SubsequenceMedium

Find the length of the longest common subsequence of two strings.

Python
def longestCommonSubsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n+1) for _ in range(m+1)]
    for i in range(1, m+1):
        for j in range(1, n+1):
            if text1[i-1] == text2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1  # match
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
Time: O(m×n)Space: O(m×n)

Greedy Algorithms ⭐ Most Important

Locally OptimalNo BacktrackIntervals
▼

A Greedy Algorithm makes the locally optimal choice at each step with the hope of finding the global optimum — and commits to that choice without ever reconsidering.

When does Greedy work? (Both must hold):

  • Greedy Choice Property: A globally optimal solution can be constructed by making locally optimal choices.
  • Optimal Substructure: Optimal solution to the whole = greedy choice + optimal solution to the remainder.

Key Mindset: "What's the single best decision I can make RIGHT NOW, and will committing to it never hurt me later?"

Always Verify! Greedy doesn't always work. Classic counterexample: coin change with arbitrary denominations (need DP). Always prove or test your greedy intuition.

Interval Scheduling

Sort by end time. Always pick activity that ends earliest. Leaves most room for future.

Interval Merging

Sort by start time. Merge overlapping intervals by extending end.

Fractional Knapsack

Sort items by value/weight ratio. Take highest density first.

Jump Game

Track farthest reachable index. If current index exceeds it, stuck.

Gas Station

If total gas ≥ total cost, solution exists. Start fresh when tank goes negative.

Huffman Coding

Min-heap: combine two smallest frequencies into a new node repeatedly.

Python — Activity Selection (Sort by End Time)
def activitySelection(activities):
    activities.sort(key=lambda x: x[1])  # sort by end time
    count, last_end = 0, float('-inf')
    for start, end in activities:
        if start >= last_end:
            count += 1
            last_end = end
    return count

#435Non-overlapping IntervalsMedium

Find the minimum number of intervals to remove so the rest don't overlap.

Python
def eraseOverlapIntervals(intervals):
    if not intervals: return 0
    intervals.sort(key=lambda x: x[1])  # sort by end
    count, last_end = 0, intervals[0][1]
    for i in range(1, len(intervals)):
        if intervals[i][0] < last_end:  # overlap!
            count += 1  # remove this one
        else:
            last_end = intervals[i][1]
    return count

Greedy: when overlap occurs, remove the interval that ends later (keep the one with smaller end — it's less obstructive to future intervals).

Time: O(n log n)Space: O(1)
#55Jump GameMedium

Each element is the max jump length from that position. Can you reach the last index?

Python
def canJump(nums):
    max_reach = 0
    for i, jump in enumerate(nums):
        if i > max_reach:
            return False  # can't reach here
        max_reach = max(max_reach, i + jump)
    return True
Time: O(n)Space: O(1)
